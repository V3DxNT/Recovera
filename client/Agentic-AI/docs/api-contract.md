# Agent Integration Contract

## Public API
The Agentic AI brain layer exposes exactly one function to Person 1.

\`\`\`typescript
import { runAgent, toAuditLogEntry } from "@/Agentic-AI/agent/index";
import type { AgentInput, DiagnosticReport, AuditLogEntry } from "@/Agentic-AI/agent/index";

const report: DiagnosticReport = await runAgent(input);
\`\`\`

## What Person 1 Provides
- An `AgentInput` payload matching the TypeScript definition.
- `incident_status`: Determines execution path ("pending", "running", "done", "failed").
- `incident_id`: Hash generated by Person 1 to uniquely identify the incident.

## What Person 1 Reads
- `report.decision_path`: Defines whether to execute the fix ("auto_fix"), ask for approval ("approval_required"), or just alert ("alert_only").
- `report.action_taken`: Maps to Person 1's AWS execution helpers.
- `report.requires_human_review`: Gatekeeper for auto-execution and UI indicators.
- `report.summary`: 2-sentence description for the UI.
- `report.notification`: Pre-formatted `SlackPayload` for `chat.postMessage`.
- `report.risk_score`: Value for the dashboard.
- `toAuditLogEntry(report, input)`: Generates an `AuditLogEntry` for storage.

## Forbidden Actions
- Do not import `llm-caller.ts`, `rca.ts`, or any other internal module.
- Do not bypass the agent for auto-fixes.
